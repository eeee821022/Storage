## 📝 如何使用本檢查表

### 新建工具時
1. 複製此檢查表
2. 在開發過程中逐項檢查
3. 確保所有項目都符合規範

### 檢查現有工具時
1. 開啟工具的 HTML 檔案
2. 對照檢查表逐項確認
3. 標記不符合的項目並修正

### 團隊協作時
1. 將此檢查表加入專案文檔
2. Code Review 時使用此表檢查
3. 確保所有成員遵循相同規範

## 🔍 快速檢查方法

### 視覺檢查
打開 `index.html` 和三個工具頁面，並排顯示：
- 主標題字體和顏色是否一致？
- 按鈕樣式是否相同？
- 配色方案是否統一？
- 圓角和陰影是否一致？

### 程式碼檢查
在每個 HTML 檔案中搜尋：
- `font-family` - 確認使用 Inter 和 Noto Sans TC
- `.action-button` - 確認按鈕樣式一致
- 顏色代碼（如 #ffffff、#2563eb）- 確認使用規範中的顏色

### 瀏覽器開發者工具檢查
1. 按 F12 開啟開發者工具
2. 選取元素檢查其樣式
3. 對照規格書確認數值

## ✨ 設計一致性的重要性

維持設計一致性可以：
- ✅ 提升使用者體驗（用戶感覺在使用同一套工具）
- ✅ 降低學習成本（操作邏輯相同）
- ✅ 建立專業形象（視覺統一、精緻）
- ✅ 簡化維護工作（遵循同一套規範）
- ✅ 提高開發效率（有明確的設計標準）

---

**記住：一致性比完美更重要！** 即使無法 100% 符合所有細節，保持整體風格的統一性仍然是最重要的。

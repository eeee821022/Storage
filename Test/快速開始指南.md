## 🎯 三個工具的使用時機

### 🗺️ 顧客旅程地圖
**什麼時候用？**
- 要了解用戶完整體驗流程
- 需要視覺化用戶情緒變化
- 想找出體驗痛點和機會點

**範例情境：**
「我們想知道用戶從認識產品到購買的完整旅程，以及每個階段的感受。」

### 🌸 曼陀羅思考法
**什麼時候用？**
- 需要從一個主題發散思考
- 想要系統化地探索問題
- 要產生大量創意想法

**範例情境：**
「我們要為新產品發想功能，需要從多個角度思考可能性。」

### 📋 親和圖法
**什麼時候用？**
- 有大量零散的資料或想法
- 需要分類整理訪談結果
- 要從混亂中找出規律

**範例情境：**
「我們訪談了 20 位用戶，現在要整理這些意見並歸納出主要需求。」

## 💾 資料儲存說明

### 重要提醒
⚠️ **所有工具都是在瀏覽器本地運作**，資料儲存在您的電腦上。

### 如何儲存工作成果？

每個工具都提供兩種儲存方式：

1. **匯出 JSON**（推薦）
   - 點擊工具列的「匯出 JSON」按鈕
   - 將工作內容儲存為 `.json` 檔案
   - 下次可透過「匯入 JSON」恢復工作

2. **截圖**
   - 點擊「截圖」按鈕
   - 將當前畫面儲存為 PNG 圖片
   - 適合分享給團隊或放入簡報

### 建議的工作流程

```
1. 使用工具進行思考/整理
   ↓
2. 定期點擊「匯出 JSON」備份
   ↓
3. 完成後使用「截圖」功能保存視覺化結果
   ↓
4. 將 JSON 檔案和截圖一起存檔
```

## 🔧 常見問題

### Q: 可以離線使用嗎？
A: 可以！下載檔案後，即使沒有網路也能使用。但初次開啟需要網路載入 Tailwind CSS 和字型。

### Q: 資料會上傳到雲端嗎？
A: 不會。所有資料都儲存在您的瀏覽器本地，不會傳送到任何伺服器。

### Q: 可以多人協作嗎？
A: 目前不支援即時協作。但您可以：
- 匯出 JSON 分享給團隊成員
- 他們可以匯入 JSON 繼續編輯
- 或使用截圖功能分享成果

### Q: 如何更新檔案名稱？
A: 請參考 **`檔案重新命名指南.md`**，裡面有詳細的步驟說明。

### Q: 如何分享給團隊？
A: 有三種方式：
1. **檔案分享**：將整個資料夾壓縮後傳送
2. **GitHub Pages**：部署到線上，分享網址（參考部署指南）
3. **成果分享**：使用截圖功能，將結果匯出成圖片

## 💡 使用技巧

### 技巧 1：右鍵標記
在曼陀羅思考法工具中，對任意格子**按右鍵**可以標記顏色，幫助您分類想法！

### 技巧 2：JSON 版本控制
為您的 JSON 檔案加上日期命名，例如：
- `product-idea-2024-11-16.json`
- `user-journey-v1.json`

### 技巧 3：組合使用
三個工具可以搭配使用：
1. 用**曼陀羅**發想功能
2. 用**親和圖**整理歸類
3. 用**旅程地圖**規劃流程
